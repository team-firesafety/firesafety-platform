## Project Overview

This is "소담 (소방 업무를 담다)" - an AI-powered fire safety integrated platform with Korean language database columns. The platform provides 4 core features: data visualization, document generation, fire risk prediction, and conversational AI.

## Architecture

### Backend (FastAPI)
- **Location**: `backend/`
- **Entry point**: `backend/app/main.py`
- **Key architecture**: Function-based routing system where all features are accessed through `/chat` endpoint
- **Database**: PostgreSQL with Korean column names (e.g., `사망자수`, `재산피해액`)
- **AI Integration**: OpenAI GPT-4o for natural language to SQL conversion and function routing

### Frontend (Vue.js 3 + Vite)
- **Location**: `frontend/`
- **Framework**: Vue 3 with Composition API
- **Build tool**: Vite
- **Key libraries**: Chart.js for visualizations, Axios for API calls

### Database Schema
- All database tables use Korean column names for better readability
- Main tables: `seoul_fire_dispatch`, `seoul_forest_fire_dispatch`, `seoul_vehicle_fire_dispatch`, `seoul_rescue_dispatch`, `national_fire_status`
- Schema defined in: `backend/migrations/visualization_schema.sql`

## Development Commands

### Backend Setup
```bash
cd backend
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
pip install -r requirements.txt

# Run development server
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Frontend Setup
```bash
cd frontend
npm install

# Development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

### Database Setup
```bash
# Create database
psql -U YOUR_USERNAME -c "CREATE DATABASE fire_safety_db;"

# Run migrations
psql -U YOUR_USERNAME -d fire_safety_db -f backend/migrations/visualization_schema.sql

# Load initial data (required for visualization features)
cd backend
python data_loader.py

# Build RAG vectorstore for chat functionality
python -m backend.scripts.build_vectorstore
```

## Key Service Architecture

### Function Router System
The platform uses a centralized function routing system in `backend/app/services/function_caller.py`:
- All user queries go through `/chat` endpoint
- GPT-4o determines which of 4 features to use based on query
- Each feature has dedicated service modules in `backend/app/services/features.py`

### Korean Database Integration
- All database operations use Korean column names
- Column mapping service: `backend/app/services/column_mapping_service.py`
- Natural language to SQL conversion happens in visualization service
- Data loader automatically converts English CSV columns to Korean database columns

### Four Core Features
1. **Data Visualization** (functionNo: 1): Natural language to SQL with Korean columns
2. **Document Generation** (functionNo: 2): PDF generation for fire safety documents
3. **Fire Risk Prediction** (functionNo: 3): Building-level risk assessment with map data
4. **Conversational AI** (functionNo: 4): RAG-based chat using FAISS vectorstore
